<objective>
Add `vault-cli lint` command that detects and optionally fixes common frontmatter issues in task files.
</objective>

<context>
Project: ~/Documents/workspaces/vault-cli (Go CLI, cobra, Obsidian markdown)
Read CLAUDE.md and existing code before making changes.

The `vault-cli list` command currently emits these warnings from real vault files:

1. `no frontmatter found` - file has no --- delimiters
2. `cannot unmarshal !!str 'high' into int` - priority field is string ("high", "must", "should") instead of int
3. `cannot unmarshal !!str 'medium' into int` - same issue with priority
4. `mapping key "assignee" already defined` - duplicate YAML key in frontmatter

Task frontmatter schema (valid):
```yaml
---
status: todo          # string: todo, in_progress, backlog, completed, hold, aborted
page_type: task       # string
priority: 1           # int (1=highest)
assignee: bborbe      # string (no duplicates)
goals:
  - "[[Goal Name]]"   # list of strings
---
```
</context>

<requirements>
1. Add `lint` subcommand in pkg/cli/cli.go:
   - `vault-cli lint` — scan all tasks, report issues (exit 1 if any found)
   - `vault-cli lint --fix` — auto-fix fixable issues, report unfixable ones

2. Add LintOperation in pkg/ops/lint.go:
   Interface: LintOperation with Execute(ctx, vaultPath, fix bool) error

3. Detect these issues (per file):
   - MISSING_FRONTMATTER: file has no frontmatter block
   - INVALID_PRIORITY: priority field is string ("high", "must", "should", "medium", "low") instead of int
   - DUPLICATE_KEY: duplicate YAML key (e.g. assignee defined twice)
   - INVALID_STATUS: status value not in allowed set

4. Fix logic for --fix:
   - INVALID_PRIORITY: map strings → int (high/must=1, medium/should=2, low=3), write file
   - DUPLICATE_KEY: keep first occurrence, remove duplicates, write file
   - MISSING_FRONTMATTER: report only, cannot auto-fix
   - INVALID_STATUS: report only, cannot auto-fix

5. Output format:
   ```
   WARN  path/to/Task.md: INVALID_PRIORITY priority is "high", expected int
   WARN  path/to/Task.md: DUPLICATE_KEY "assignee" defined multiple times
   ERROR path/to/Task.md: MISSING_FRONTMATTER no frontmatter block found
   FIXED path/to/Task.md: INVALID_PRIORITY priority "high" → 1
   ```

6. Exit codes:
   - 0: no issues (or all issues fixed)
   - 1: issues found (unfixed)

7. Add Ginkgo tests for lint operation
8. Add counterfeiter mock for LintOperation
</requirements>
