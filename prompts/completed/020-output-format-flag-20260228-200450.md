<objective>
Add `--output` flag to all vault-cli commands supporting `plain` (default) and `json` output formats.
This enables machine-readable output for task-orchestrator integration.
</objective>

<context>
Go CLI project at ~/Documents/workspaces/vault-cli.
Read CLAUDE.md for project conventions.
Read ~/.claude/docs/go-patterns.md for interface/constructor patterns.

The flag must be a persistent flag on the root command so it applies to all subcommands automatically.
Output is currently printed directly in ops layer (pkg/ops/*.go) via fmt.Printf/fmt.Println.
</context>

<requirements>
1. Add `--output` persistent flag to root command in `./pkg/cli/cli.go`:
   - Values: `plain` (default) or `json`
   - Persistent so all subcommands inherit it automatically

2. Pass output format down to all ops via their Execute/Run signatures or a shared output writer.
   Preferred approach: add `OutputFormat string` to each op's Execute method signature, OR
   pass an `io.Writer` + format to a shared printer helper.

3. Implement JSON output for each command type:

   **list commands** (task list, goal list, theme list, objective list, vision list):
   - plain: current `[status] Name` format
   - json: array of objects with all available fields:
     ```json
     [{"name": "Task Name", "status": "in_progress", "assignee": "bborbe", "priority": -1, "vault": "personal"}]
     ```

   **mutation commands** (complete, defer, work-on, update):
   - plain: current emoji + message format
   - json: `{"success": true, "name": "Task Name", "vault": "personal"}`
   - on error: `{"success": false, "error": "error message"}`

   **lint commands**:
   - plain: current WARN/ERROR/FIXED format
   - json: array of issues `[{"file": "path", "type": "WARN", "description": "..."}]`

   **search commands**:
   - plain: current one-result-per-line format
   - json: array of strings `["result1", "result2"]`

4. Multi-vault output with `--output json`: merge all vault results into a single array.
   Add `"vault": "vault-name"` field to each item so caller knows which vault it came from.
   No `=== vault-name ===` headers in JSON mode.

5. Warnings (e.g. "Warning: failed to update goal") in JSON mode:
   Include in response as `"warnings": ["..."]` field instead of printing to stdout.
   Always print warnings to stderr in plain mode.
</requirements>

<implementation>
Suggested shared helper in `./pkg/cli/output.go` (new file):

```go
package cli

import (
    "encoding/json"
    "fmt"
    "os"
)

const (
    OutputFormatPlain = "plain"
    OutputFormatJSON  = "json"
)

func printJSON(v any) error {
    enc := json.NewEncoder(os.Stdout)
    enc.SetIndent("", "  ")
    return enc.Encode(v)
}
```

Root command flag registration:
```go
var outputFormat string
rootCmd.PersistentFlags().StringVar(&outputFormat, "output", "plain", "output format: plain or json")
```

Pass `outputFormat` into each command's RunE closure and down to ops Execute calls.

For mutation ops, change signature example:
```go
func (o *completeOperation) Execute(ctx context.Context, vaultPath, taskName string) (CompleteResult, error)
```
Where `CompleteResult` contains fields needed for both plain and JSON output.
</implementation>
