<objective>
Add `vault-cli task validate <task-name>` command that runs lint on a single task by name
and exits with non-zero code if issues found.
Enables task-orchestrator to validate a specific task directly.
</objective>

<context>
Go CLI project at ~/Documents/workspaces/vault-cli.
Read CLAUDE.md for project conventions.
Read ~/.claude/docs/go-patterns.md for interface/constructor patterns.

`vault-cli task lint` already exists — it lints ALL tasks in a vault directory.
`task validate` is a focused variant: find task by name, lint only that file, return issues.
The existing LintOperation in `pkg/ops/lint.go` works on a directory.
Storage already supports `FindTaskByName` to get the task file path.
</context>

<requirements>
1. Add `vault-cli task validate <task-name>` subcommand in `./pkg/cli/cli.go`.
   - Args: exactly one argument (task name)
   - Respects `--vault` flag and `--output` flag

2. Reuse existing lint logic from `pkg/ops/lint.go` but scope it to a single file:
   - Find task by name via storage.FindTaskByName
   - Run lint checks on that single file only
   - No `--fix` flag needed for validate (read-only)

3. Exit codes:
   - 0 → no issues found
   - 1 → lint issues found (WARN or ERROR)

4. plain output:
   ```
   WARN  My Task.md: missing_field status is empty
   ERROR My Task.md: invalid_status "wip" is not a valid status
   ```
   If no issues: `✅ My Task: no lint issues found`

5. json output:
   ```json
   {
     "name": "My Task",
     "vault": "personal",
     "issues": [
       {"type": "WARN", "issue_type": "missing_field", "description": "status is empty"},
       {"type": "ERROR", "issue_type": "invalid_status", "description": "\"wip\" is not a valid status"}
     ]
   }
   ```
   If no issues: `{"name": "My Task", "vault": "personal", "issues": []}`

6. Multi-vault: if --vault not set, search all vaults for task by name.
   If found in multiple vaults, validate in each and combine results.

7. Write tests in `./pkg/ops/` — check if existing lint test file covers single-file variant,
   add tests if not. Use real temp dirs (os.MkdirTemp pattern from go-testing.md).
</requirements>

<implementation>
Simplest approach — add a method or reuse LintOperation scoped to one file path:

```go
// In cli.go RunE:
vaults, err := getVaults(ctx, configLoader, vaultName)
for _, vault := range vaults {
    storageConfig := storage.NewConfigFromVault(vault)
    store := storage.NewStorage(storageConfig)

    task, err := store.FindTaskByName(ctx, vault.Path, taskName)
    if err != nil {
        continue // not in this vault
    }

    // Run lint on the single task file path
    issues, err := lintOp.ExecuteFile(ctx, taskFilePath)
    // ... print issues
}
```

If LintOperation doesn't support single-file, add `ExecuteFile(ctx, filePath) ([]LintIssue, error)` method.
</implementation>
